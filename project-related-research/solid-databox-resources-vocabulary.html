This vocabulary defines the classes and properties used by Solid-Databox [[SOLID-DATABOX-PROTOCOL]] to catalogue the resources — servers, applications, libraries, vocabularies, specifications, developers, communities, and hosted Pod providers — that make up the Solid, Read-Write-Web (RWW), and broader Semantic-Web ecosystem. The records are harvested from four CSV inventories kept under dev-resources/ and re-published as Linked Data. The ontology reuses schema.org [[SCHEMA-ORG]], FOAF [[FOAF]], DOAP [[DOAP]], SKOS [[SKOS]], DCAT [[DCAT]], PROV-O [[PROV-O]], Dublin Core [[DCTERMS]], and LDP [[LDP]], coining the sdr: prefix only where those leave a gap.

Unofficial draft, no standing at any standards body, published to structure work toward the Solid 2026 Hackathon. Companion to the [[SOLID-DATABOX-VOCABULARY]] and [[SOLID-DATABOX-PROTOCOL]]. The namespace https://w3id.org/solid-databox/resources# (prefix sdr:) is provisional.

This document defines classes and properties, and a small number of usage rules for them. Where those rules use RFC 2119 keywords they bind any implementation that produces or consumes Solid-Databox resource records.

Design Principles

  1. Reuse over reinvention. Anchor on schema.org, FOAF, DOAP, SKOS, DCAT, PROV-O, DCTERMS, and LDP; coin sdr: terms only for genuine gaps — chiefly the CSV provenance columns.
  2. The CSV is the source of truth. Every sdr:Resource SHOULD carry sdr:sourceFile and the relevant CSV column properties so the record can be regenerated deterministically.
  3. One record per row. Each non-empty row in a source CSV becomes exactly one RDF resource; duplicate URLs across CSVs are merged with owl:sameAs.
  4. People are first-class. A developer/luminary is typed as both sdr:Developer and foaf:Person / schema:Person, so existing profile tooling resolves them.

Classes

sdr:Resource

Subclass of: schema:CreativeWork, dcat:Resource

The top-level catalogue class. Every record harvested from a CSV row is an instance of sdr:Resource or one of its subclasses.

sdr:SoftwareResource

Subclass of: sdr:Resource, schema:SoftwareApplication, doap:Project

A piece of software — server, app, library, utility, or platform.

sdr:Server

Subclass of: sdr:SoftwareResource

A server implementation that hosts Solid Pods or implements the Solid protocol stack (e.g. Community Solid Server, Node Solid Server, ESS, Trinpod, Manas, PHP Solid Server, OpenLink Virtuoso).

sdr:App

Subclass of: sdr:SoftwareResource, schema:SoftwareApplication

An end-user application built on Solid or the Read-Write-Web.

sdr:Library

Subclass of: sdr:SoftwareResource

A reusable library, SDK, or framework for building Solid / Semantic-Web software.

sdr:Utility

Subclass of: sdr:SoftwareResource

A developer utility, CLI tool, browser extension, or integration helper.

sdr:Platform

Subclass of: sdr:SoftwareResource

A broader platform or framework that incorporates Solid / RWW capabilities.

sdr:Vocabulary

Subclass of: sdr:Resource; owl:equivalentClass skos:ConceptScheme, owl:Ontology

An RDF vocabulary, ontology, or namespace (e.g. schema.org, FOAF, SKOS, ODRL, PROV-O). Sourced primarily from Tools - commonVocab.csv.

sdr:Specification

Subclass of: sdr:Resource

A W3C / community specification underpinning the Solid or Semantic-Web stack (e.g. LDP, SHACL, WebID-TLS, Solid Protocol, VC Data Model).

sdr:Developer

Subclass of: sdr:Resource, foaf:Person, schema:Person

A person who contributes to the ecosystem. The CSVs use the label Luminary for notable contributors; skos:altLabel preserves that term.

sdr:Community

Subclass of: sdr:Resource; owl:equivalentClass foaf:Group, schema:Organization

A community group, company, or organisation active in the ecosystem.

sdr:PodProvider

Subclass of: sdr:Community

An organisation that offers hosted Solid Pods to end users (e.g. Inrupt Pod Spaces, solidcommunity.net, trinpod.eu, use.id).

sdr:ResourceListing

Subclass of: sdr:Resource

A curated list or directory of resources (e.g. an "awesome" list or the solidproject.org/apps catalogue).

Properties

sdr:sourceFile

Domain: sdr:Resource · Range: xsd:string

The CSV file within dev-resources/ from which the record was harvested (e.g. Tools - RWW_Solid Links (mostly github).csv).

sdr:csvType

Domain: sdr:Resource · Range: xsd:string

The value of the Type column (e.g. Solid, RWW, SemWeb, News). Values are also enumerated in the sdr:TypeScheme SKOS concept scheme.

sdr:csvCategory & sdr:csvSubCategory

Domain: sdr:Resource · Range: xsd:string

The Category and Sub-Category CSV columns, preserved verbatim for traceability.

sdr:implementationLanguage

Domain: sdr:SoftwareResource · Range: xsd:string

Subproperty of: schema:programmingLanguage

The primary programming language of a software resource.

sdr:licenseType

Domain: sdr:Resource · Range: xsd:string

Subproperty of: dct:license

The license under which a resource is distributed (e.g. MIT, GPL, Commercial).

sdr:maintenanceStatus

Domain: sdr:Resource · Range: xsd:string

A human-readable maintenance status (e.g. actively maintained, legacy, experimental).

sdr:hostingLocation

Domain: sdr:PodProvider · Range: xsd:string

Subproperty of: schema:areaServed

The geographic region in which a Pod provider hosts Pods (e.g. EU, US, Australia).

sdr:hasDeveloper

Domain: sdr:SoftwareResource · Range: sdr:Developer

Subproperty of: schema:author, dct:creator

Links a software resource to a person who develops or maintains it.

sdr:hasCommunity

Domain: sdr:Resource · Range: sdr:Community

Subproperty of: schema:publisher, dct:publisher

Links a resource to the community or organisation that maintains it.

sdr:usesVocabulary

Domain: sdr:SoftwareResource · Range: sdr:Vocabulary

Subproperty of: dct:references

Links a software resource to a vocabulary/ontology it depends on or emits.

sdr:implementsSpecification

Domain: sdr:SoftwareResource · Range: sdr:Specification

Subproperty of: dct:references

Links a software resource to a specification it implements.

sdr:prefix & sdr:namespaceURI

Domain: sdr:Vocabulary

The conventional prefix (e.g. foaf) and namespace URI of a vocabulary. sdr:namespaceURI is a subproperty of schema:url.

sdr:purpose

Domain: sdr:Vocabulary · Range: xsd:string

Subproperty of: dct:description

A short statement of the purpose of a vocabulary (from the commonVocab CSV).

SKOS Concept Scheme — CSV "Type" values

The Type column of the source CSVs is enumerated as a SKOS concept scheme, sdr:TypeScheme, with top concepts:

  • sdr:TypeSolid — Solid: resources directly part of the Solid ecosystem.
  • sdr:TypeRWW — RWW: Read-Write-Web resources, the precursor movement.
  • sdr:TypeSemWeb — SemWeb: general Semantic-Web / Linked-Data resources.
  • sdr:TypeNews — News: news articles and press coverage about Solid.

Example — a Server record

@prefix sdr:     .
@prefix schema:  .
@prefix dct:     .
@prefix doap:    .


    a sdr:Server , schema:SoftwareApplication , doap:Project ;
    dct:title         "Community Solid Server" ;
    dct:description   "An open and modular implementation of the Solid specifications." ;
    sdr:sourceFile    "Tools - RWW_Solid Links (mostly github).csv" ;
    sdr:csvType       "Solid" ;
    sdr:csvCategory   "Server" ;
    sdr:implementationLanguage "TypeScript" ;
    sdr:licenseType   "MIT" ;
    sdr:maintenanceStatus "actively maintained" ;
    schema:url         ;
    doap:repository    ;
    sdr:hasCommunity   .

Example — a Vocabulary record

@prefix sdr:  .
@prefix dct:  .


    a sdr:Vocabulary ;
    dct:title       "Schema Org" ;
    sdr:prefix      "schema" ;
    sdr:namespaceURI "https://schema.org/" ;
    sdr:purpose     "SEO" ;
    sdr:sourceFile  "Tools - commonVocab.csv" .

Example — a Developer / Luminary record

@prefix sdr:   .
@prefix foaf:  .
@prefix schema:  .


    a sdr:Developer , foaf:Person , schema:Person ;
    foaf:name       "Tim Berners-Lee" ;
    sdr:sourceFile  "Tools - RWW_Solid Links (mostly github).csv" ;
    sdr:csvType     "Solid" ;
    sdr:csvCategory "Luminary" ;
    schema:url       .

Source CSVs

The four CSV inventories under dev-resources/ that this ontology catalogues:

  1. Solid Resources - Links (mostly github).csv — the original mixed Solid / RWW / SemWeb link list.
  2. Tools - RWW_Solid Links (mostly github).csv — the enriched, language-tagged Solid / RWW tool list.
  3. Tools - commonVocab.csv — RDF vocabularies and ontologies.
  4. Tools - SemWeb-tool lists.csv — broad Semantic-Web tools, specifications, vendors, and plugins.
